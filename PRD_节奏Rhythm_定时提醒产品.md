# 产品需求文档（PRD）：节奏 Rhythm —— 灵活间隔提醒助手

> 版本：v1.0 ｜ 状态：评审稿 ｜ 作者：产品团队 ｜ 日期：2026-07-20

---

## 1. 产品概述

### 1.1 一句话定位
**「节奏」是一款"每隔 n 分钟，提醒你做任何事"的轻量习惯助手**——把护眼、喝水、起身、拉伸、吃药、番茄专注，甚至"站起来接个电话"这类任意微习惯，统一成一套可配置、懂场景、不打扰的提醒引擎。

### 1.2 背景与问题
- 长期屏幕工作者普遍面临眼疲劳、久坐、缺水、注意力涣散等问题，医学上有明确建议节奏（如 20-20-20 护眼法则、每小时起身 2–3 分钟）。
- 但用户"知道该做 ≠ 记得做"。现有工具分散且单一：护眼 App 只管眼、喝水 App 只管水、番茄钟只管专注，**没有一款产品让用户用同一套引擎管理"所有周期性小动作"**。
- 用户痛点是"提醒太机械、太打扰"：开会时弹提醒、投屏时弹提醒、专注时被打断，导致用户大量"跳过/关闭"，习惯无法养成。

### 1.3 产品目标
- **核心目标**：让用户"设置一次，长期受益"，把健康/效率类微习惯自动化。
- **差异化目标**：通用间隔引擎 + 模板化场景 + 场景感知免打扰（智能暂停）。
- **商业目标**：通过免费高频健康提醒获客，高级模板/统计/多端同步变现。

### 1.4 平台策略（已决策：桌面端优先，预留移动端）
- **首发形态**：桌面端（Windows / macOS），以"菜单栏/系统托盘常驻 + 全局快捷键 + 全屏休息弹窗"为核心交互，参考一休、xBreak、Stretchly 验证过的桌面健康场景（付费意愿强、后台保活与空闲检测更可靠）。
- **技术选型建议**：跨平台桌面框架（Tauri/Electron），业务逻辑用 Rust/TS 沉淀为**与平台无关的"间隔引擎"核心模块**。
- **移动端预留**：iOS/Android 后期通过复用同一套间隔引擎核心 + 云端配置同步接入，不重复造轮子；PRD 中 P1 的"云同步 + 多端"即为移动端预留的接口与数据层。
- **UX 说明**：当前配套原型以移动交互逻辑做演示，正式开发将 re-skin 为桌面窗口 + 菜单栏迷你态（见 §6 信息架构补充）。

## 2. 市场与竞品

### 2.1 竞品矩阵（已调研）
| 产品 | 平台 | 核心场景 | 间隔能力 | 智能免打扰 | 统计/游戏化 | 商业模式 |
|---|---|---|---|---|---|---|
| Stand Up! | iOS | 起身站立 | 固定 45min | 弱 | 进度追踪 | 免费+内购 |
| Eye Care 20-20-20 | iOS | 护眼 | 固定 20min | 弱 | 无 | 免费+内购 |
| StretchMinder | iOS | 拉伸/呼吸 | 可配 | 弱 | 无 | 免费+内购 |
| NatureTimer | Web | 护眼/水/番茄 | 可配（5min–8h） | 工作模式 | 无 | 免费 |
| Break Timer（Chrome） | 扩展 | 水/姿势/番茄 | 20/30/45/60min | 空闲检测+暂停 | 7天图+连胜 | 免费 |
| 一休（macOS） | 桌面 | 护眼/水/站立 | 可配+番茄 | 白名单/投屏暂停/勿扰 | 次数统计 | 付费 |
| xBreak（macOS） | 桌面 | 护眼/久坐/番茄 | 可配+强制锁屏 | 弱 | 状态监测 | 付费 |
| Restio（iOS） | 移动 | 护眼+眼部锻炼 | 可配 | 弱 | 经验/徽章/好友 | 免费+高级版 |

### 2.2 机会点与差异化
1. **通用间隔引擎**：不只预设"护眼/喝水"，而是"任何事 + 任意 n 分钟"，降低用户认知门槛，覆盖长尾场景（吃药、滴人工泪液、深呼吸、喝水、背单词……）。
2. **场景感知免打扰（核心壁垒）**：基于设备空闲检测、投屏/会议状态、应用白名单、工作时间窗，自动暂停或延迟提醒——解决"被打扰就关掉"的留存杀手。
3. **模板化冷启动**：内置科学模板（WHO/AAO 建议），新用户 30 秒建好第一个提醒，降低上手成本。
4. **游戏化+社交**：连胜、成就、好友监督（参考 Restio），提升长期留存。
5. **多端一致**：移动为主、桌面/浏览器扩展为辅，习惯配置云端同步。

## 3. 目标用户与场景

### 3.1 用户画像
- **主要（P0）**：22–40 岁知识工作者/学生，日均屏幕 6h+，有健康焦虑但执行力弱。
- **次要（P1）**：远程办公者、慢性病患者（定时服药）、备考/备考群体（番茄专注）。

### 3.2 核心场景
| 场景 | 典型需求 | 间隔 |
|---|---|---|
| 护眼 | 每 20min 看远 20 秒 | 20min |
| 喝水 | 每小时喝一杯水 | 60min |
| 防久坐 | 每 45min 起身活动 | 45min |
| 拉伸 | 每 90min 做肩颈操 | 90min |
| 服药 | 每 8h 服药 | 480min |
| 番茄专注 | 25min 专注 + 5min 休息 | 25/5min |
| 自定义 | "每 30min 站起来接个电话" | 任意 |

## 4. 功能需求

### 4.1 功能清单与优先级
**P0（MVP）**
- F1 提醒模板库（护眼/喝水/起身/拉伸/服药/番茄/自定义）
- F2 通用间隔配置（n 分钟，1–1440 可选；支持"工作/休息"双时长，如番茄）
- F3 提醒弹窗（全屏休息态 + 倒计时 + 完成/跳过/延后）
- F4 通知渠道（系统通知 + 提示音 + 震动，可预览）
- F5 自定义提醒文案与图标
- F6 工作时间窗（仅工作时段提醒）
- F7 一键暂停/恢复
- F8 基础统计（今日完成次数、跳过次数、连续天数）

**P1（增强）**
- F9 场景感知免打扰：设备空闲检测、投屏/会议自动暂停、应用白名单
- F10 预告倒计时（休息前 10 秒预留保存/暂停）
- F11 严格模式（隐藏跳过，需通关动作才能跳过）
- F12 多维统计看板（日/周/月趋势、热力图）
- F13 引导式动作（护眼操/拉伸 Gif/Lottie 动画）
- F14 云同步 + 多端（iOS/Android/桌面/扩展）

**P2（远期）**
- F15 游戏化：经验/等级/徽章/好友监督
- F16 AI 智能排程（按作息/日历自动建议提醒节奏）
- F17 小组件/灵动岛/手表实时活动
- F18 开放模板社区（UGC 分享提醒模板）

### 4.2 关键功能详述
- **间隔引擎**：底层是"触发器 = 周期(n) + 时间段(工作窗) + 状态闸门(免打扰)"。支持单时长与双时长（番茄）。
- **提醒弹窗（休息态）**：全屏柔和界面，大字号文案 + 圆形倒计时进度 + 三个动作【完成 ✓ / 延后 5min / 跳过】；可配背景动画、提示音、是否锁屏（强制休息）。
- **场景感知免打扰**：检测键盘鼠标空闲 >X 分钟视为离开，自动暂停；检测到投屏/会议软件前台自动暂停；白名单 App 前台不弹。该能力是留存关键，需算法低误判。
- **统计**：以"完成率"和"连续天数"为核心激励指标。

## 5. 交互流程（用户旅程）

```
[冷启动] 打开 App → 推荐模板卡片（护眼/喝水/起身）→ 点选「护眼 20-20-20」→ 一键启用
   ↓
[日常] 状态栏/悬浮显示下次提醒倒计时 → 到点弹全屏休息 → 用户完成/跳过
   ↓
[管理] 首页查看所有提醒及今日进度 → 新建/编辑/暂停/删除
   ↓
[复盘] 统计页查看周趋势与连续天数 → 获得徽章激励 → 持续使用
```

关键交互原则：**默认克制不打扰；任何打断都给"延后/跳过"出口；重要工作可一键全局暂停。**

## 6. 信息架构与页面清单

```
节奏 Rhythm
├─ 首页（提醒列表 + 今日进度环 + 全局暂停）
├─ 新建/编辑提醒（模板选择 → 间隔/文案/声音/免打扰 → 保存）
├─ 提醒弹窗（全屏休息态，倒计时 + 动作）
├─ 统计（日/周/月趋势、连续天数、徽章）
└─ 设置（工作时间窗、智能免打扰、主题、云同步、关于）
```

UX 原型覆盖：首页、新建提醒、提醒弹窗、统计、设置 共 5 个核心页面（见配套 HTML 原型）。

**桌面端形态补充（已决策，见 §1.4）**：桌面端在以上页面之外，需额外设计两种常驻态——
- **菜单栏/托盘迷你态**：常驻显示"距下次提醒倒计时 + 一键暂停/跳过"的最小控件（参考一休状态栏倒计时、BreakCycle 菜单栏 popover），日常无感守护。
- **全屏休息弹窗**：到点时覆盖全屏（或锁定屏幕强制休息），即上述"提醒弹窗"，支持多显示器场景（仅主屏/所有屏）。
- 注：当前 HTML 原型以移动交互逻辑演示上述 5 个页面，正式 UX 将 re-skin 为桌面窗口 + 菜单栏迷你态，交互逻辑一致。

## 7. 非功能需求
- **通知可靠性**：后台保活/系统通知通道（iOS 通知扩展、Android 前台服务、桌面托盘），到点必达，误差 < 5s。
- **性能**：App 冷启动 < 1.5s；提醒触发 CPU 占用可忽略；常驻内存 < 80MB。
- **多端一致性**：配置云端同步，冲突以"最后修改"为准。
- **隐私**：健康/习惯数据本地优先，云端加密；明示不出售数据（参考竞品隐私承诺）。
- **无障碍**：支持动态字体、高对比、屏幕阅读器。

## 8. 数据指标
- **北极星指标**：周活跃用户周均"提醒完成次数"。
- **留存**：次日 / 7 日 / 30 日留存目标 ≥ 45% / 30% / 18%。
- **激活**：新用户 24h 内启用 ≥ 1 个提醒的比例 ≥ 70%。
- **习惯养成**：连续 21 天启用提醒用户占比（核心健康价值证明）。
- **变现**：高级版付费转化率（P1 后）目标 ≥ 3%。

---

## 9. 商业模式
- **免费层**：基础提醒（任意间隔）、系统通知、基础统计、单端使用。
- **高级版（订阅）**：云同步多端、智能免打扰全功能、高级统计/热力图、引导式动画库、AI 排程、去广告。
- **潜在 B 端**：企业版（员工健康关怀、工位久坐提醒批量部署）。

## 10. 里程碑 / 迭代规划
- **M1（4 周）**：MVP——F1–F8，**桌面端优先（Win/macOS）**：菜单栏/托盘常驻 + 全局快捷键 + 全屏休息弹窗 + 模板冷启动 + 基础统计。间隔引擎核心模块独立封装，为移动端预留。
- **M2（4 周）**：P1 核心——F9–F13 场景感知免打扰 + **云同步（移动端预留接口）** + 引导动画。
- **M3（4 周）**：P2 增长——F15 游戏化 + **iOS/Android 接入（复用引擎核心）** + 桌面小组件/灵动岛等价物。
- **M4+**：AI 排程、模板社区、企业版。

## 11. 风险与开放问题
- **系统通知限制**：iOS 后台触发受限，需评估"本地通知 + 常驻小组件"方案。
- **免打扰误判**：投屏/会议检测在不同 OS 可行性不同，M2 需做技术验证。
- **留存 vs 打扰**：严格模式提升完成率但降低体验，需 A/B 测试平衡。
- **开放问题（已决策）**：~~首发平台选 iOS 还是 Android？是否先做桌面端？~~ → **已定：桌面端优先（Win/macOS），移动端（iOS/Android）通过共享间隔引擎核心 + 云同步在 P1 阶段预留接入**，见 §1.4。
